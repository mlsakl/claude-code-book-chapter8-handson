## アーキテクチャパターン

### レイヤードアーキテクチャ

```
┌─────────────────────────────────────────┐
│  CLIレイヤー（src/cli/）                  │  ← ユーザー入力の受付・バリデーション・表示
├─────────────────────────────────────────┤
│  サービスレイヤー（src/services/）         │  ← ビジネスロジック（TaskManager, GitService）
├─────────────────────────────────────────┤
│  データレイヤー（src/storage/）            │  ← JSONファイルへの読み書き（FileStorage）
└─────────────────────────────────────────┘
             ↕
     .task/tasks.json（ローカルファイル）
```

#### CLIレイヤー（`src/cli/`）
- **責務**: コマンド定義、ユーザー入力の受付、バリデーション、結果の表示
- **許可される操作**: サービスレイヤー（TaskManager, GitService）の呼び出し
- **禁止される操作**: FileStorage（データレイヤー）への直接アクセス、ビジネスロジックの実装

#### サービスレイヤー（`src/services/`）
- **責務**: ビジネスロジック（タスクCRUD、Gitブランチ管理、ステータス遷移）
- **許可される操作**: FileStorage（データレイヤー）の呼び出し、simple-gitの使用
- **禁止される操作**: CLIレイヤーへの依存、ターミナル出力の直接実施

#### データレイヤー（`src/storage/`）
- **責務**: `.task/tasks.json` の読み書き、バックアップ管理
- **許可される操作**: Node.js fsモジュールへのアクセス
- **禁止される操作**: ビジネスロジックの実装、外部APIの呼び出し

## データ永続化戦略

### ストレージ方式

| データ種別 | ストレージ | フォーマット | 理由 |
|-----------|----------|-------------|------|
| タスクデータ | ローカルファイル（`.task/tasks.json`） | JSON | シンプル、Git管理可能、外部依存なし |
| 設定データ | ローカルファイル（`.task/config.json`） | JSON | 軽量、人が読める形式 |
| バックアップ | ローカルファイル（`.task/backup/`） | JSON | ファイルコピーで実現、復元が簡単 |

### バックアップ戦略

- **タイミング**: 書き込み操作（create/update/delete）の直前に自動バックアップ
- **保存先**: `.task/backup/tasks.json.<ISO8601タイムスタンプ>.bak`
- **世代管理**: 最新10世代を保持（超過分は古いものから自動削除）
- **手動バックアップ**: `task backup` コマンドで任意のタイミングで実行可能
- **復元方法**: バックアップファイルを `.task/tasks.json` にコピーして上書き

### アトミック書き込み

データ破損を防ぐため、書き込みは以下の手順で実施:

1. `.task/tasks.json.tmp` に新データを書き込む
2. 書き込み成功を確認
3. `.task/tasks.json.tmp` を `.task/tasks.json` にリネーム（OSレベルのアトミック操作）

## パフォーマンス要件

### レスポンスタイム

| 操作 | 目標時間 | 測定環境 |
|------|---------|---------|
| `task add` | 100ms以内 | 標準的な開発用PC、SSD |
| `task list`（100件） | 100ms以内 | 標準的な開発用PC、SSD |
| `task list`（1000件） | 1秒以内 | 標準的な開発用PC、SSD |
| `task start`（Gitブランチ作成） | 2秒以内 | 標準的な開発用PC |
| `task show` | 100ms以内 | 標準的な開発用PC、SSD |

### リソース使用量

| リソース | 上限 | 理由 |
|---------|------|------|
| メモリ（常駐） | 50MB | CLIツールとしての軽量性を維持 |
| メモリ（1万件処理時） | 100MB | JSONを全てメモリに展開するため |
| ディスク（`.task/`） | 10MB以内（通常使用） | タスクデータ＋バックアップ10世代 |

## セキュリティアーキテクチャ

### データ保護

- **暗号化**: 機密データ（GitHubトークン）は暗号化なし、代わりにファイルパーミッション `600` で保護
- **アクセス制御**: `.task/config.json` は作成時に `chmod 600` を適用
- **機密情報管理**: GitHubトークンは `.task/config.json` に保存し、`.gitignore` に自動追加

### 入力検証

- **バリデーション対象**: タイトル（1〜200文字）、ステータス値（列挙型）、優先度（列挙型）、日付（YYYY-MM-DD形式）、タスクID（UUID形式または部分一致）
- **サニタイゼーション**: ファイルパス操作には `path.join()` を使用してパストラバーサルを防止
- **エラーハンドリング**: エラーメッセージにはシステム内部情報（スタックトレース等）を含めず、ユーザーフレンドリーなメッセージのみ表示

### コマンドインジェクション防止

- Git操作は `simple-git` ライブラリのAPIのみを使用し、シェルへの直接コマンド実行（`exec`/`spawn`）は行わない

## スケーラビリティ設計

### データ増加への対応

- **想定最大データ量**: タスク1万件（JSONファイル〜5MB程度）
- **パフォーマンス劣化対策**: 1万件超過時に起動時に警告を表示し、SQLite移行を推奨するメッセージを出す
- **アーカイブ戦略**: `task status <id> archived` でアーカイブ状態に移行し、`task list` のデフォルト表示から除外

### 機能拡張性

- **GitHub連携（P1）**: `GitHubService` クラスをサービスレイヤーに追加するだけで既存コードへの影響を最小化
- **SQLite移行（将来）**: `FileStorage` と同じインターフェースを実装する `SQLiteStorage` を追加し、切り替えはコンストラクタの差し替えのみで対応
- **設定のカスタマイズ**: `.task/config.json` で拡張可能な設定管理

## テスト戦略

### ユニットテスト

- **フレームワーク**: Jest + ts-jest
- **対象**: `TaskManager`（CRUD、ステータス遷移）、`FileStorage`（読み書き）、`GitService`（ブランチ名生成）、バリデーション関数
- **カバレッジ目標**: 80%以上
- **モック**: FileStorageをモック化してTaskManagerをテスト、simple-gitをモック化してGitServiceをテスト

### 統合テスト

- **方法**: 実際のファイルシステムを使用（テスト用一時ディレクトリ）
- **対象**: `task init` → `task add` → `task start` → `task done` のフルフロー、フィルタリング・ソート動作

### E2Eテスト

- **ツール**: `child_process.exec` で実際のCLIコマンドを実行し、終了コードと標準出力を検証
- **シナリオ**: 基本操作フロー、エラーケース（Gitなし環境での `task start`）、パフォーマンス（1000件）

## 技術的制約

### 環境要件

- **OS**: macOS 12+、Ubuntu 22.04+、Windows 10+（Git Bash推奨）
- **Node.js**: v18.0.0以上（v24.11.0推奨）
- **最小メモリ**: 512MB（実行環境）
- **必要ディスク容量**: インストール30MB程度 + データ領域
- **必要な外部依存**: Git（`git` コマンドが PATH に含まれること）

### パフォーマンス制約

- JSONファイルを全てメモリに展開するため、タスク1万件超過時はメモリ使用量が増加
- Gitコマンドの実行時間はリポジトリサイズに依存し、大規模リポジトリでは `task start` が遅くなる可能性がある

### セキュリティ制約

- GitHubトークンをコードにハードコードすることを禁止（`.task/config.json` または環境変数 `TASKCLI_GITHUB_TOKEN` を使用）
- `.task/config.json` を `.gitignore` に追加しないと、トークンがGitに誤ってコミットされるリスクがある（`task init` で自動追加）
